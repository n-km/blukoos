We've successfully established the foundational build system for BlukoOS, achieving the following key milestones:

Build System & Toolchain

  • Target Architecture: The kernel is compiled for the 32-bit x86 architecture (`-m32 -m elf_i386`), ensuring compatibility with older and virtualized hardware.
  • Development Environment: We utilize GCC as both the C compiler and assembler, with specific flags like -ffreestanding and -nostdlib to create an independent, bare-metal operating system kernel.
  • Bootloader Integration: The system is set up to be booted by the GRUB Multiboot bootloader. We generate a bootable ISO file using grub-mkrescue.

Initial Kernel Components

The kernel's current structure includes foundational modules necessary for basic operation:

  • Core Kernel: kernel/kernel.c serves as the main entry point for the OS.
  • VGA Driver: kernel/drivers/video/vga.c for text-mode output to the screen.
  • Keyboard Driver: kernel/drivers/input/keyboard.c for handling user input.
  • Console Management: kernel/console/console.c for abstracting screen output and input handling.

Execution & Testing

  • Virtualization: We use QEMU (`qemu-system-i386`) to reliably test and run the generated bootable ISO in a controlled virtual environment.
  • Build Commands: The Makefile provides clear commands for building (make all), running (make run), and cleaning the project (make clean).
